01 — Structure

Anatomie d'un script NEXUS

Un script est un fichier .py ordinaire avec trois parties obligatoires : une docstring de métadonnées, une fonction principale, et un point d'entrée CLI. 

┌─────────────────────────────────────────────────────────────┐
│  1. DOCSTRING  — lue par NEXUS pour configurer les ports    │
└─────────────────────────────────────────────────────────────┘
"""
@workflow:name:         Nom de la brique
@workflow:description:  Ce que fait ce script.
@workflow:input:        { … }
@workflow:output:       { … }
@workflow:dependencies: package1,package2
"""

┌─────────────────────────────────────────────────────────────┐
│  2. IMPORTS (standards uniquement au niveau module)         │
└─────────────────────────────────────────────────────────────┘
import os, json
from pathlib import Path

┌─────────────────────────────────────────────────────────────┐
│  3. FONCTION PRINCIPALE  — reçoit dict, retourne dict       │
└─────────────────────────────────────────────────────────────┘
def ma_brique(params: dict) -> dict:
    ...
    return {"sortie": valeur}

┌─────────────────────────────────────────────────────────────┐
│  4. POINT D'ENTRÉE CLI  — pour tester sans NEXUS            │
└─────────────────────────────────────────────────────────────┘
if __name__ == "__main__":
    import sys
    data = json.loads(sys.argv[1]) if len(sys.argv) > 1 else {}
    print(json.dumps(ma_brique(data), ensure_ascii=False, indent=2))
⚠️
La docstring doit être le tout premier élément du fichier — avant tout import. NEXUS la lit sans exécuter le code.
02 — Docstring

Format de la docstring @workflow

Cinq balises reconnues par NEXUS. Les blocs @workflow:input et @workflow:output sont du JSON strict.

"""
@workflow:name: Conversion PDF en Images           Titre affiché dans le nœud

@workflow:description: Convertit chaque page d'un
    PDF en images PNG ou JPEG.                      Description (libre)

@workflow:input: {
    "pdf_path":   {"type": "string", "required": true,  "description": "Chemin du PDF"},
    "output_dir": {"type": "string", "default": "./output", "description": "Dossier de sortie"},
    "format":     {"type": "list",   "options": ["PNG","JPEG"], "default": "PNG"},
    "dpi":        {"type": "int",    "min": 72, "max": 600, "default": 150}
}

@workflow:output: {
    "images": {"type": "array",  "description": "Chemins des images générées"},
    "count":  {"type": "int",    "description": "Nombre d'images produites"}
}

@workflow:dependencies: pdf2image,Pillow          pip install — séparés par virgules
"""
03 — Entrées

Types de paramètres d'entrée

Chaque clé de @workflow:input génère un port et un widget dans le nœud NEXUS.

string
Texte libre, chemin, URL, clé API…
default · required
"default": "valeur"
int
Entier avec bornes min/max optionnelles.
min · max · default
"min": 1, "max": 100, "default": 10
double
Décimal (float). Coefficient, score, ratio…
min · max · default
"default": 1.5
bool
Vrai / Faux. Sélecteur dans le nœud.
default (true ou false)
"default": false
list
Choix parmi une liste fixe (menu déroulant).
options · default
"options": ["PNG","JPEG","TIFF"]
file
Chemin d'un fichier. Bouton 📂 dans le nœud.
default
Python reçoit un dict :
chemin_complet, fichier, emplacement
folder
Chemin d'un dossier. Bouton 📂 dans le nœud.
default
Python reçoit un str
"default": "./output"

Lire les paramètres dans la fonction

Utilisez toujours .get() avec une valeur par défaut — NEXUS peut ne pas envoyer toutes les clés.

def ma_brique(params: dict) -> dict:
    # string
    texte      = params.get("texte",      "")

    # int / double — cast explicite conseillé
    nombre     = int(params.get("nombre",    10))
    coeff      = float(params.get("coeff",    1.0))

    # bool — JSON envoie True/False Python
    actif      = bool(params.get("actif",    True))

    # list → string (valeur choisie dans le menu)
    format_out = params.get("format_out", "PNG")

    # file → dict avec chemin_complet / fichier / emplacement
    finfo      = params.get("fichier",    {})
    chemin     = finfo.get("chemin_complet", "") if isinstance(finfo, dict) else finfo

    # folder → string
    dossier    = params.get("dossier",   "./output")
💡
Imports lourds dans la fonction — placez import numpy, from PIL import Image, etc. à l'intérieur de la fonction. NEXUS parse la docstring sans créer d'environnement d'exécution.
04 — Sorties

Types de retour (@workflow:output)

Le dict retourné doit contenir exactement les clés déclarées dans @workflow:output.

string
Texte, chemin, message de statut.
"resultat": "texte produit"
int
Compteur, code retour, nombre d'éléments.
"count": 42
float
Score, durée, ratio, pourcentage.
"score": 0.97
bool
Succès / échec du traitement.
"succes": True
array
Liste Python (strings, dicts, nombres…).
"items": ["a", "b", "c"]
object
Dictionnaire Python arbitraire.
"meta": {"key": "val"}
# Les clés du return doivent correspondre EXACTEMENT à @workflow:output
return {
    "images":     liste_chemins,     # array
    "count":      len(liste_chemins),  # int
    "output_dir": dossier_out,       # string
}
🚫
Pas de print() parasite. NEXUS capture stdout pour lire le JSON retourné. Tout affichage hors du bloc __main__ fera échouer le parsing. Utilisez sys.stderr.write() pour les logs de débogage.
05 — Exemple complet

Script fonctionnel — scripts/modele.py

Exemple sans dépendance externe. Testez avec python scripts/modele.py.

"""
@workflow:name: Transformateur de Données
@workflow:description: Transforme et répète un texte selon des options.
@workflow:input: {
    "texte":       {"type": "string", "default": "Bonjour !",           "required": true,  "description": "Texte source"},
    "repetitions": {"type": "int",    "min": 1, "max": 20, "default": 3,               "description": "Nombre de répétitions"},
    "majuscules":  {"type": "bool",   "default": false,                                "description": "Mettre en majuscules"},
    "separateur":  {"type": "list",   "options": [" | "," / "," — "],  "default": " | ", "description": "Séparateur"},
    "dossier_out": {"type": "folder", "default": "./output",                            "description": "Dossier de sortie"}
}
@workflow:output: {
    "texte_final":   {"type": "string", "description": "Texte transformé"},
    "mots":          {"type": "array",  "description": "Mots uniques du texte source"},
    "nb_caracteres": {"type": "int",    "description": "Longueur du texte final"},
    "fichier":       {"type": "string", "description": "Chemin du fichier résultat"}
}
@workflow:dependencies:
"""

import os, json, re
from pathlib import Path
from datetime import datetime


def transformateur_de_donnees(params: dict) -> dict:
    # ── Lecture des paramètres ────────────────────────────────────
    texte       = params.get("texte",       "Bonjour !")
    repetitions = int(params.get("repetitions", 3))
    majuscules  = bool(params.get("majuscules",  False))
    separateur  = params.get("separateur",  " | ")
    dossier_out = params.get("dossier_out", "./output")

    # ── Validation ───────────────────────────────────────────────
    if not texte.strip():
        raise ValueError("Le paramètre 'texte' ne peut pas être vide.")

    # ── Traitement ───────────────────────────────────────────────
    occurrence   = texte.upper() if majuscules else texte
    texte_final  = separateur.join([occurrence] * repetitions)
    mots         = sorted(set(re.findall(r'\b\w+\b', texte.lower())))

    # ── Écriture fichier ─────────────────────────────────────────
    Path(dossier_out).mkdir(parents=True, exist_ok=True)
    fichier = os.path.join(dossier_out, f"resultat_{datetime.now().strftime('%H%M%S')}.txt")
    Path(fichier).write_text(texte_final, encoding="utf-8")

    # ── Retour ────────────────────────────────────────────────────
    return {
        "texte_final":   texte_final,
        "mots":          mots,
        "nb_caracteres": len(texte_final),
        "fichier":       fichier,
    }


if __name__ == "__main__":
    import sys
    data = json.loads(sys.argv[1]) if len(sys.argv) > 1 else {}
    print(json.dumps(transformateur_de_donnees(data), ensure_ascii=False, indent=2))
06 — Checklist

Avant de charger dans NEXUS

La docstring est le tout premier élément du fichier (avant tout import)
Les blocs JSON (@workflow:input / @workflow:output) sont valides — guillemets doubles, pas de virgule finale
Les clés du return correspondent exactement aux noms dans @workflow:output
Les imports lourds (numpy, PIL, torch…) sont dans la fonction, pas au niveau module
Aucun print() hors du bloc __main__
Test CLI réussi : python mon_script.py '{}' → affiche un JSON valide
Dépendances listées dans @workflow:dependencies (séparées par virgules)
